Distinguish between Verification and Validation in the V-Model.

---

- **Verification (Left side):** A static process (reviews, walkthroughs, code review) evaluating if "we are building the product right" without running the software.
- **Validation (Right side):** A dynamic process (executing code with actual test cases) evaluating if "we are building the right product" according to user requirements.
