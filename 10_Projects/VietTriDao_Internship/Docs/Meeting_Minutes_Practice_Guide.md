# 📓 Vở Bài Tập Thực Hành: Viết Biên Bản Họp (Meeting Minutes) Tiếng Anh

## TL;DR

Tài liệu thiết kế dưới dạng **Vở bài tập thực hành (Workbook)** giúp luyện viết biên bản họp (Meeting Minutes) chuẩn Agile. Gồm cấu trúc, từ vựng theo phần, ví dụ thực tế (Trước/Sau) và 3 bài tập thực hành thực tế.

---

## 📋 Phần 1: Khung Mẫu Tái Sử Dụng (Reusable Template)

_Copy đoạn code dưới đây làm khung sườn mỗi khi viết biên bản họp nhóm:_

```markdown
# MEETING MINUTES - [PROJECT NAME]

**Date & Time:** YYYY-MM-DD | HH:MM - HH:MM
**Location:** [e.g. Google Meet / Office]
**Secretary (Người ghi chép):** [Name]
**Attendees:** [Member 1, Member 2...]

## 1. Meeting Agenda

- [Agenda item 1]
- [Agenda item 2]

## 2. Key Decisions

- **[Decision 1]:** Description...
- **[Decision 2]:** Description...

## 3. Action Items

| Task / User Story  | Assigned To | Deadline   | Status |
| :----------------- | :---------- | :--------- | :----- |
| [Task description] | [Name]      | YYYY-MM-DD | Todo   |

## 4. Next Meeting

- **Date & Time:** YYYY-MM-DD | HH:MM
- **Agenda:** [Topic to discuss]
```

## 🔑 Phần 2: Từ Vựng & Mẫu Câu Theo Từng Phần

### 1. Header & Agenda (Thông tin chung & Chủ đề)

- **Từ vựng:** _Facilitator_ (người điều phối), _Secretary/Scribe_ (thư ký/người ghi chép), _Attendees_ (người tham gia), _Agenda_ (chủ đề họp).
- **Mẫu câu:**
  - _“The meeting was called to order at [Time] by [Name].”_ (Cuộc họp bắt đầu lúc... do... điều phối).
  - _“The primary objective of today’s meeting is to [V-inf]...”_ (Mục tiêu chính của buổi họp hôm nay là để...).

### 2. Discussion Points (Nội dung thảo luận)

- **Từ vựng:** _Discuss_ (thảo luận), _Propose/Suggest_ (đề xuất), _Tradeoffs_ (ưu/nhược điểm), _Highlight_ (nhấn mạnh).
- **Mẫu câu:**
  - _“Nam proposed using ReactJS for the frontend due to its component-based architecture.”_ (Nam đề xuất dùng ReactJS cho FE vì kiến trúc component của nó).
  - _“The team discussed the tradeoffs between SQL and NoSQL databases.”_ (Cả team đã thảo luận về ưu/nhược điểm giữa cơ sở dữ liệu SQL và NoSQL).

### 3. Key Decisions (Quyết định chính)

- **Từ vựng:** _Agree on_ (đồng thuận), _Decide to_ (quyết định làm gì), _Finalize_ (chốt lại).
- **Mẫu câu:**
  - _“It was agreed that Nam will take the role of Team Leader.”_ (Đã thống nhất Nam sẽ nhận vai trò Nhóm trưởng).
  - _“The team finalized the database schema design.”_ (Cả team đã chốt thiết kế sơ đồ database).

### 4. Action Items (Phân công công việc)

- **Từ vựng:** _Assign_ (phân công), _Be responsible for / Be in charge of_ (chịu trách nhiệm cho), _Set a deadline_ (đặt hạn chót).
- **Mẫu câu:**
  - _“Nam will be in charge of setting up the GitHub repository.”_ (Nam sẽ chịu trách nhiệm tạo repo GitHub).
  - _“The deadline for the Figma design is set to June 25th.”_ (Hạn chót cho thiết kế Figma được đặt vào ngày 25/06).

## 🧩 Phần 3: Ví Dụ Thực Tế (Before & After)

### Bối cảnh (Trước khi chuẩn hóa - Ghi chép thô)

> _"Họp chiều nay lúc 1h30 đến 2h. Có Nam, Vy, Tuấn. Nam làm trưởng nhóm nha. Vy vẽ Figma màn hình đăng nhập, ráng xong trước ngày mai nha Vy. Tuấn thì thiết kế database PostgreSQL, vẽ ERD, hạn chót ngày kia 26/06. Nam tạo github add mọi người vô."_

### Biên bản chuẩn hóa (Sau khi chuẩn hóa - Professional English)

```markdown
# MEETING MINUTES - TEAM KICK-OFF

**Date & Time:** 2026-06-24 | 13:30 - 14:00
**Secretary:** Nam
**Attendees:** Nam, Vy, Tuấn

## 1. Meeting Agenda

- Select the Team Leader.
- Assign roles and initial tasks.
- Setup workspace.

## 2. Key Decisions

- Nam was selected as the Team Leader.
- The team chose PostgreSQL as the primary database system.

## 3. Action Items

| Task / User Story                           | Assigned To | Deadline   | Status |
| :------------------------------------------ | :---------- | :--------- | :----- |
| Set up GitHub repository and invite members | Nam         | 2026-06-24 | Todo   |
| Draw Figma mockup for Login screen          | Vy          | 2026-06-25 | Todo   |
| Design Database Schema (ERD)                | Tuấn        | 2026-06-26 | Todo   |
```

## 📝 Phần 4: Bài Tập Thực Hành (Practice Drills)

_Mày hãy thực hiện 3 bài tập dưới đây để tự kiểm tra và luyện viết._

### Bài tập 1: Dịch Biên Bản Họp (Translation Drill)

**Đề bài:** Hãy dịch ghi chép cuộc họp sau đây sang tiếng Anh chuyên nghiệp:

> _"Lan đề xuất dùng framework ExpressJS cho Back-end. Tuấn đồng ý nhưng khuyên nên viết bằng TypeScript để dễ kiểm soát lỗi. Cả team chốt dùng Node.js + ExpressJS + TypeScript. Tuấn chịu trách nhiệm viết base code trước 27/06. Lan phụ trách viết tài liệu API doc hạn chót 28/06."_

- **Chỗ viết bài làm của mày:**
  - _Luyện tập:_ ...
- **Gợi ý đáp án (Bôi đen để xem):**
  > _It was agreed that the backend will be developed using Node.js, ExpressJS, and TypeScript. Tuấn will be in charge of setting up the base code by 2026-06-27. Lan is responsible for writing the API documentation with a deadline of 2026-06-28._

### Bài tập 2: Chuẩn Hóa Đoạn Chat Nhóm (Chat Log to Minutes Drill)

**Đề bài:** Hãy chuyển cuộc trò chuyện lộn xộn trong group chat Zalo sau thành bảng **Action Items** bằng tiếng Anh:

> - _Vy (14:05):_ "Mọi người ơi, ai vẽ Figma cho màn hình Dashboard bệnh nhân thế? Mình đang bận làm màn hình Login rồi."
> - _Tuấn (14:06):_ "Để mình làm cho, chiều mai mình gửi bản nháp nhé."
> - _Nam (14:07):_ "Ok Tuấn. Mà Vy nhớ test thử giao diện responsive trên mobile cho màn hình Login nha, hạn chót ngày kia 26/06."
> - _Lan (14:08):_ "Mình BA sẽ hỗ trợ Tuấn làm rõ các trường thông tin hiển thị trên Dashboard chiều nay luôn."

- **Chỗ viết bài làm của mày (Bảng Action Items):**

  | Task / User Story | Assigned To | Deadline | Status |
  | :---------------- | :---------- | :------- | :----- |
  | ...               | ...         | ...      | ...    |

- **Gợi ý đáp án:**

  | Task / User Story                           | Assigned To | Deadline   | Status |
  | :------------------------------------------ | :---------- | :--------- | :----- |
  | Prepare draft of Patient Dashboard Figma    | Tuấn        | 2026-06-25 | Todo   |
  | Test mobile responsiveness for Login screen | Vy          | 2026-06-26 | Todo   |
  | Clarify data fields for Patient Dashboard   | Lan         | 2026-06-24 | Todo   |

### Bài tập 3: Phát Hiện Lỗi Biên Bản (Error Identification Drill)

**Đề bài:** Tìm 3 lỗi nghiêm trọng trong đoạn biên bản họp sau (về mặt quy trình Agile/Scrum) và sửa lại:

> - **Decisions:** Quyết định sẽ làm việc thật nhanh để kịp tiến độ.
> - **Action Items:**
>   - Code API cho toàn bộ hệ thống (Nam làm, hạn chót: khi nào xong thì thôi).
>   - Viết testcase (Lan làm).

- **Chỗ viết bài làm của mày (Chỉ ra lỗi và sửa lại):**
  - _Lỗi 1:_ ... -> _Sửa:_ ...
  - _Lỗi 2:_ ... -> _Sửa:_ ...
  - _Lỗi 3:_ ... -> _Sửa:_ ...
- **Gợi ý đáp án:**
  1. _Lỗi 1:_ Quyết định quá mơ hồ, không đo lường được ("làm việc thật nhanh"). -> _Sửa:_ Đưa ra quyết định công nghệ rõ ràng (ví dụ: Chốt sử dụng ReactJS và PostgreSQL).
  2. _Lỗi 2:_ Không có deadline rõ ràng cho task code API ("khi nào xong thì thôi"). -> _Sửa:_ Đặt ngày cụ thể (ví dụ: Deadline: 2026-06-28).
  3. _Lỗi 3:_ Task "Viết testcase" quá lớn, chung chung và thiếu người chịu trách nhiệm rõ ràng cùng deadline. -> _Sửa:_ Phân rã công việc (WBS) thành: _"Viết testcase cho module Authentication"_ (Assign: Lan, Deadline: 2026-06-25).
